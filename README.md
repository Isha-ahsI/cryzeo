=======================================
Cryzeo - React Cryptocurrency Platform
=======================================

Welcome to Cryzeo, a modern cryptocurrency platform built with React 18 and Bootstrap 5. Experience real-time crypto tracking, market analysis, and seamless trading with our responsive and feature-rich application.

## ✨ Key Features

- 📊 Real-time cryptocurrency price tracking
- 📈 Interactive charts with Chart.js
- 🎨 Modern UI with Bootstrap 5 components
- 🚀 Blazing fast performance with React 18
- 📱 Fully responsive design for all devices
- 🎭 Smooth animations with Framer Motion
- 🔍 SEO optimized structure
- 💰 Multiple crypto-related pages and features

🚀 Getting Started
---------------------------------------
1. Clone the repository:
   ```bash
   git clone [your-repository-url].git
   cd cryzeo
   ```

2. Install dependencies:
   ```bash
   npm install
   # or
   yarn install
   ```

3. Start the development server:
   ```bash
   npm run dev
   # or
   yarn dev
   ```

4. Access the application:
   - Open [http://localhost:3000](http://localhost:3000) in your browser
   - View the application in development mode
   - Hot reload enabled for instant updates

5. Build for production:
   ```bash
   npm run build
   npm start
   ```

🛠️ Project Structure & Customization
---------------------------------------
### Pages & Routing
- Uses React Router DOM for client-side routing
- Main pages include:
  - Home (`/`)
  - About (`/about`)
  - Blog (`/blog`)
  - Prices (`/prices`)
  - Contact (`/contact-us`)

### Styling
- **Bootstrap 5** as the main CSS framework
- **SASS/SCSS** for custom styles
- Organized in a modular structure:
  - `scss/components/` - Component-specific styles
  - `scss/layout/` - Layout and grid systems
  - `scss/pages/` - Page-specific styles

### Components
- **Reusable UI Components** in `src/components/`
  - Homepage sections
  - Navigation and footer
  - Cards and modals
  - Form elements
  - Charts and data visualization

### Data Management
- Static data in component files
- API routes for dynamic content
- Environment variables for configuration
